# v1.0.2 Development Checklist

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Deliver a more automation-friendly and easier-to-maintain Weibo evidence workflow on top of `v1.0.1`, with stronger structured outputs, cleaner control flow, and better submission verification.

**Architecture:** Keep `script/weibo_manual_comment_flow.py` as the single entry point, but reduce duplication in `main()` by introducing a shared run-state / finish helper. Extend the current regression-first workflow by adding narrow unit tests before behavior changes, then add a small doc update when the new flags or outputs land.

**Tech Stack:** Python 3.12, Playwright sync API, Pillow, pytest

## Task 1: Refactor run finalization into a shared helper

**Objective:** Remove repeated `emit_final_summary(...)`, `page.screenshot(...)`, `browser.close()`, and `sys.exit(...)` branches in `main()`.

**Files:**
- Modify: `script/weibo_manual_comment_flow.py:475-786`
- Test: `tests/test_evidence_output.py`
- Test: `tests/test_p0_regressions.py`

**Why this is in v1.0.2:**
The current `main()` repeats near-identical failure-exit logic across multiple branches (`login-required-initial`, `login-required-after-submit`, `submission-not-confirmed`, `comment-not-found`, `comment-found-no-box`). That makes future changes to summary fields and artifact behavior risky.

**Checklist:**
- [ ] Introduce a helper such as `finalize_run(...)` or `finish_with_status(...)`
- [ ] Centralize summary emission, optional screenshot capture, and return/exit code mapping
- [ ] Replace ad-hoc branch-local `emit_final_summary(...)` blocks with the helper
- [ ] Preserve existing status names and exit codes unless intentionally changed

**Verification:**
- Run: `pytest -q tests/test_evidence_output.py tests/test_p0_regressions.py`
- Expected: all pass
- Run: `python3 -m py_compile script/weibo_manual_comment_flow.py`
- Expected: no output

## Task 3: Strengthen submission verification semantics

**Objective:** Improve `verify_comment_submission(...)` so v1.0.2 confirms a fresher, more attributable comment signal than “visible matching text exists somewhere under comments root”.

**Files:**
- Modify: `script/weibo_manual_comment_flow.py:294-329, 572-626, 636-690`
- Test: `tests/test_p0_regressions.py`
- Test: `tests/test_reliability_regressions.py`

**Observed baseline:**
Current verification checks for a visible comment text match in the comment root. That is safer than whole-page matching, but still does not distinguish a newly submitted comment from an older duplicate.

**Checklist:**
- [ ] Add a helper that prefers fresh list state after `按时间`
- [ ] Record which selector/root produced the verification signal
- [ ] Extend submission evidence to include richer verification detail, e.g.:
  - `verification_mode`
  - `matched_selector`
  - `time_sort_confirmed`
- [ ] Preserve a boolean `comment_visible` for backward compatibility while adding more fields

**Verification:**
- Run: `pytest -q tests/test_p0_regressions.py tests/test_reliability_regressions.py`
- Expected: all pass
- Add at least one regression test proving the richer verification payload exists
